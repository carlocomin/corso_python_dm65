🎛️ Getter, Setter e @property in Python

🔹 Perché usare getter e setter?

  • Per proteggere l’accesso diretto agli attributi interni
  • Per aggiungere logica durante lettura/scrittura (es. validazione)
  • In Python si usa il decoratore @property

🔸 Esempio classico con @property

class Conto:
    def __init__(self, saldo):
        self._saldo = saldo

    @property
    def saldo(self):
        return self._saldo

    @saldo.setter
    def saldo(self, valore):
        if valore < 0:
            raise ValueError("Il saldo non può essere negativo")
        self._saldo = valore

c = Conto(100)
print(c.saldo)      # ➜ 100
c.saldo = 200       # setter chiamato
c.saldo = -50       # ➜ ValueError

🔹 Alternative senza property (non raccomandato)

class Persona:
    def __init__(self):
        self._nome = "Mario"

    def get_nome(self):
        return self._nome

    def set_nome(self, valore):
        self._nome = valore

✔ Con @property si mantiene una sintassi semplice, pur aggiungendo controllo e protezione.
